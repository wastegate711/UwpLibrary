# Библиотека для работы с микросхемой pcf8575, работает так же, как pcf8574

`Initialization(int address, I2cBusSpeed busSpeed = I2cBusSpeed.StandardMode)` - Получает I2C контроллер по умолчанию в системе (в Raspberry pi 2 и 3 это I2С1) есть перегрузка с указанием названия контроллера и если нужно обрабатывать прерывания, то можно указать номер gpio порта.

`event Action<byte[]> PinChanged` - Событие генерируется на любом восходящем или нисходящем фронте входов порта микросхемы Interrupt.

`Dispose()` -  Удаляет объекты и закрывает дескрипторы.

`SetPinState(ushort pin)` - Управляет состоянием выводов микросхемы.

`byte[] ReadPinState()` - Считывает состояние выводов микросхемы. Возвращает состояние всех выводов в виде массива байт, где нулевой элемент массива
это порты от 0 до 7, а первый элемент содержит порты от 10 до 17.

`ResetState()` - Делает сброс в исходное состояние.

# Пример
Для работы нужно получить дескриптор устройства подключенного к I2C, если устройства с указанным адресом не найдено инициализация вернет `false`.
Регистрируем в контейнере класс и интерфейс или создаем экземпляр класса.

`IPcf8575 _pcf8575; 
 bool init = aweit _pcf8575.Initialization(0x20);`
Далее можно изменять состояние выводов или считывать состояние.
Если нужно обрабатывать прерывание микросхемы, тогда нужно подписаться на событие.
`_pcf8575.PinChanged += Foo();`
